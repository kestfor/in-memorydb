VersionManager управляет версиями обновлений для каждой ноды, его задача - принимать решения о том, что делать с приходящими от других нод обновлениями,
для этого он хранит версии - его локальный источник правды и ссылку на engine, чтобы изменять данные через api
задача engine хранить данные, это сам движок, он не должен знать о том, как именно обновлять данные, он должен просто принимать обновления и применять их, это самое низкоуровневый слой
задача version manager принимать решения о том, что делать с обновлениями, и применять их к engine
нужна прослойка которая объединит engine и version manager
